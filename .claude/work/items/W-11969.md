## One-Liner

Core UI: display today's theme, show submissions, voting interface, winner reveal. All vanilla JS from JSON.

## Problem Statement

Users need to see the day's theme, browse submissions with song info and album art, cast votes, and see the winner — all rendered from the JSON data files.

## Proposed Solution

1. Vanilla JS reads competition `config.json` and determines current round by date
2. Phase detection from `config.json` schedule times + timezone (converted to user's local TZ for display)
3. Three phase views: submission (theme + songs), voting (ballots + Spotify previews), results (winner + tallies)
4. Song cards with title, artist, album art, submitter name
5. Leaderboard sidebar from `leaderboard.json`
6. On-brand loading state (CRT flicker / pixel font "LOADING...") and error state ("GAME OVER")

## Acceptance Criteria

Inherited from frozen Requirement W-11959 (ADO #11959):

**Scenario**: App determines current round and phase
**Given** the app loads and fetches `config.json`
**When** the current date and time are compared against the competition schedule
**Then** the correct round (`day-NN.json`) is identified
**And** the current phase (submission, voting, or results) is determined from schedule times + timezone

**Scenario**: Submission phase displays theme and songs
**Given** the current phase is submission
**When** the round view renders
**Then** today's theme is displayed prominently
**And** the theme picker's name is shown
**And** any already-submitted songs appear as cards with title, artist, album art, and submitter name

**Scenario**: Voting phase enables ballot casting
**Given** the current phase is voting
**When** the round view renders
**Then** all submissions are displayed with vote buttons
**And** a member cannot vote for their own submission
**And** song cards show title, artist, album art (from Spotify data), and submitter name

**Scenario**: Results phase reveals the winner
**Given** the current phase is results
**When** the round view renders
**Then** the winner is displayed with highlight treatment
**And** vote tallies are shown for all submissions
**And** the leaderboard sidebar updates from `leaderboard.json`

**Scenario**: Loading and error states are on-brand
**Given** the app is fetching JSON data
**When** data is loading
**Then** an on-brand loading state is shown (CRT flicker / pixel font "LOADING...")
**And** if the fetch fails, a "GAME OVER" error state is displayed

**Scenario**: Phase transitions without page reload
**Given** the user is viewing the current round
**When** the phase changes (e.g., submission → voting)
**Then** the view updates automatically via polling or timer
**And** no page reload is required

**Scenario**: Vanilla JS with responsive 80s aesthetic
**Given** the app is built with zero frameworks
**When** viewed on desktop or mobile
**Then** the layout is responsive (desktop-first, usable on mobile)
**And** the 80s arcade aesthetic is maintained across all phase views
