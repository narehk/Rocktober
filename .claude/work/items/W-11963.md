## One-Liner

Comments on submissions, emoji reactions, trash talk. GitHub Discussions/Issues API rendered in UI.

## Problem Statement

Social interaction is central to Rocktober. Without comments and reactions, it's just a voting tool. The fun comes from the banter around song choices.

## Proposed Solution

Use GitHub Discussions or Issues as the comment backend. Each round maps to a discussion/issue. Comments loaded via GitHub API and rendered in the 80s arcade UI. Emoji reactions for quick engagement.

## Acceptance Criteria

**Scenario**: Comments load from GitHub Discussions for each round
**Given** a round has an associated GitHub Discussion (linked by round number or slug)
**When** the round view renders
**Then** comments are fetched from the GitHub Discussions API
**And** each comment displays the author name, timestamp, and message body
**And** comments are rendered in the 80s arcade aesthetic

**Scenario**: Member can post a comment on a submission
**Given** a member is viewing a round with submissions
**When** they type a comment and submit it
**Then** the comment is posted to the associated GitHub Discussion via API
**And** the comment appears in the UI without a full page reload
**And** the comment is attributed to the posting member

**Scenario**: Emoji reactions on submissions
**Given** a member is viewing submissions in any phase
**When** they click a reaction emoji on a submission
**Then** the reaction is recorded via GitHub Reactions API
**And** reaction counts are displayed on each submission card
**And** multiple reaction types are supported (e.g., fire, skull, trophy, guitar)

**Scenario**: Comments render within the 80s aesthetic
**Given** comments are loaded from GitHub
**When** they are rendered in the UI
**Then** comment styling matches the CRT scanline, neon glow, and pixel font aesthetic
**And** timestamps are formatted in a retro style
**And** author names are displayed with arcade-style treatment
