## One-Liner

In-app Spotify search, submission flow. Cloudflare Worker proxy for API credentials.

## Problem Statement

Members need to search for songs within the app and submit them to the current round. The search needs to be good enough that finding the right track is easy. API credentials must be hidden from the client.

## Proposed Solution

Spotify Web API via client credentials flow, proxied through a Cloudflare Worker to hide the API secret. Search results show track name, artist, album art, and preview. Submission writes to the round JSON via GitHub API.

## Acceptance Criteria

**Scenario**: Song search returns relevant Spotify results
**Given** the app is in the submission phase and a member opens the search interface
**When** they type a song title or artist name
**Then** search results from Spotify Web API are displayed within 2 seconds
**And** each result shows track name, artist, album name, and album art thumbnail

**Scenario**: Cloudflare Worker proxies Spotify API securely
**Given** the frontend sends a search request
**When** the request reaches the Cloudflare Worker
**Then** the Worker authenticates with Spotify using client credentials flow
**And** API keys are never exposed to the client browser
**And** the Worker returns only the data fields needed by the frontend

**Scenario**: Member submits a song to the current round
**Given** a member has selected a song from search results
**When** they confirm the submission
**Then** the song is written to the current round's JSON file via GitHub API
**And** the submission includes track name, artist, album art URL, Spotify track ID, and submitter name
**And** the member sees confirmation that their submission was recorded

**Scenario**: One submission per member per round
**Given** a member has already submitted a song for the current round
**When** they attempt to submit another song
**Then** they are prompted to replace their existing submission or cancel
**And** only one submission per member exists in the round data at any time

**Scenario**: Search works without Spotify account
**Given** Spotify is used as a data source only (no user auth required)
**When** any member searches for a song
**Then** results are returned regardless of whether the member has a Spotify account
**And** the client credentials flow handles all authentication server-side
