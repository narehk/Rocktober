## One-Liner

Cron jobs: theme reveal, ballot open, vote tally, winner announcement. Commits updated JSON.

## Problem Statement

The daily cycle (theme reveal → submissions → ballot → winner) needs to run automatically. Currently done manually over Teams. GitHub Actions can automate the timing.

## Proposed Solution

GitHub Actions workflows with cron triggers: morning theme reveal, 3pm ballot open, 4:30pm vote tally and winner announcement. Each action reads/updates the competition JSON and commits changes back to the repo.

## Acceptance Criteria

**Scenario**: Theme reveal Action runs on schedule
**Given** a GitHub Actions workflow is configured with a cron trigger for theme reveal
**When** the scheduled time arrives (configurable per competition, default morning)
**Then** the Action reads the competition config and determines today's round
**And** the round JSON is updated with the day's theme and phase set to "submission"
**And** the updated JSON is committed back to the repo

**Scenario**: Ballot open Action transitions round phase
**Given** a GitHub Actions workflow is configured for ballot open
**When** the scheduled time arrives (configurable per competition, default 3pm)
**Then** the current round's phase is updated from "submission" to "voting"
**And** the updated JSON is committed back to the repo

**Scenario**: Vote tally Action computes winner and updates leaderboard
**Given** a GitHub Actions workflow is configured for vote tally
**When** the scheduled time arrives (configurable per competition, default 4:30pm)
**Then** votes in the current round are tallied
**And** the winner is recorded in the round JSON with phase set to "results"
**And** leaderboard.json is recomputed with updated standings
**And** all updated JSON files are committed back to the repo

**Scenario**: Schedule is configurable per competition
**Given** competition config.json includes schedule fields (submissionOpen, votingOpen, resultsReveal)
**When** a GitHub Action runs
**Then** it reads the competition-specific schedule from config.json
**And** cron triggers can be adjusted per competition without changing workflow code

**Scenario**: Actions handle weekends and off-days gracefully
**Given** the competition config includes a `competitionDays` field (e.g., weekdays only)
**When** a scheduled Action fires on a non-competition day
**Then** the Action exits without making changes
**And** no round is advanced or created for that day
