## One-Liner

Ballot UI, vote tally logic, leaderboard computation/display. Configurable self-vote toggle.

## Problem Statement

The daily voting and competition-wide leaderboard are core to the experience. Voting must be fair (configurable self-vote prevention), tallying must be reliable, and the leaderboard must update in real-time after each round.

## Proposed Solution

Ballot UI during voting phase shows all submissions. Votes stored in round JSON. Self-vote toggle per competition settings. Leaderboard JSON computed by GitHub Action after each round tally. Running totals across all rounds.

## Acceptance Criteria

**Scenario**: Ballot UI displays all submissions during voting phase
**Given** the current round is in the voting phase
**When** the ballot view renders
**Then** all submissions for the round are displayed as song cards
**And** each card shows track name, artist, album art, and submitter name
**And** a vote button is present on each card

**Scenario**: Self-vote prevention is configurable
**Given** a competition's config.json has a `selfVoteAllowed` setting
**When** `selfVoteAllowed` is false and a member views the ballot
**Then** the vote button is disabled on the member's own submission
**And** a visual indicator explains why they cannot vote for their own song

**Scenario**: Vote is recorded in round JSON
**Given** a member clicks the vote button on a submission
**When** the vote is confirmed
**Then** the vote is written to the round's JSON file via GitHub API
**And** the vote record includes voter name and the submission they voted for
**And** a member can only cast one vote per round

**Scenario**: Vote tallying computes correct winner
**Given** the voting phase has ended and votes are recorded in the round JSON
**When** the tally GitHub Action runs
**Then** votes are counted per submission
**And** the submission with the most votes is declared the winner
**And** the winner field is set in the round JSON

**Scenario**: Leaderboard reflects cumulative standings
**Given** multiple rounds have been completed with winners
**When** leaderboard.json is recomputed after a round
**Then** it contains each member's total wins, total submissions, and win rate
**And** members are sorted by wins descending
**And** the leaderboard sidebar in the UI updates to reflect current standings

**Scenario**: Vote counts are hidden during voting
**Given** the current round is in the voting phase
**When** a member views the ballot
**Then** vote counts are not displayed
**And** counts are only revealed after the voting phase closes and results are shown
