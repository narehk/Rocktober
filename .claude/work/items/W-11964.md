## One-Liner

Push notifications to Teams channel on key events. Extensible provider pattern for Slack/Google Chat.

## Problem Statement

Team members need to know when the theme drops, when voting opens, and who won — without checking the app constantly. Currently communicated manually over Teams.

## Proposed Solution

Teams Incoming Webhooks triggered by GitHub Actions on key events. Provider pattern abstraction so Slack, Google Chat, or other services can be added later without changing the core logic.

## Acceptance Criteria

**Scenario**: Theme reveal triggers Teams notification
**Given** the theme reveal GitHub Action has completed successfully
**When** the Action's notification step executes
**Then** a message is posted to the configured Teams channel via Incoming Webhook
**And** the message includes today's theme name, round number, and a link to the app

**Scenario**: Ballot open triggers Teams notification
**Given** the ballot open GitHub Action has completed successfully
**When** the Action's notification step executes
**Then** a message is posted to the Teams channel announcing voting is open
**And** the message includes the number of submissions and a link to vote

**Scenario**: Winner announcement triggers Teams notification
**Given** the vote tally GitHub Action has completed and a winner is determined
**When** the Action's notification step executes
**Then** a message is posted to the Teams channel with the winner's name, song, and vote count
**And** the message includes updated leaderboard standings

**Scenario**: Provider pattern supports multiple notification targets
**Given** the notification system uses an extensible provider pattern
**When** a new provider is added (e.g., Slack, Google Chat)
**Then** only a new provider module needs to be created
**And** the core notification trigger logic does not change
**And** multiple providers can be active simultaneously per competition config

**Scenario**: Webhook failure does not block the competition workflow
**Given** a Teams webhook URL is configured but the webhook call fails
**When** a notification is attempted
**Then** the failure is logged but does not prevent the GitHub Action from completing
**And** the competition phase transition still occurs
**And** the failure is reported in the Action log for debugging
