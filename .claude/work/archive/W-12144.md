## One-Liner

Eliminate Cloudflare dependency. Vote/submission persistence via GitHub Actions repository_dispatch. Song search via Spotify oEmbed (public, no auth).

## Problem Statement

The Cloudflare Worker is the single point of failure for votes, submissions, and song search. The user doesn't have a Cloudflare account and can't deploy the Worker. This blocks all testing and development of the voting/submission flow.

## Proposed Solution

1. Vote + submission persistence → GitHub Actions via `repository_dispatch`
2. Song search → Spotify oEmbed (public, credential-free) with URL paste
3. Fine-grained GitHub PAT (contents:write, single repo) for browser dispatch
4. Concurrency groups prevent race conditions on simultaneous votes

## Acceptance Criteria

- [ ] Votes persist to round JSON via GitHub Actions workflow
- [ ] Song submissions persist to round JSON via GitHub Actions workflow
- [ ] Song search works via Spotify URL paste + oEmbed (no Spotify credentials needed)
- [ ] Cloudflare account/Worker no longer required
- [ ] Concurrent votes are serialized (no lost votes)
- [ ] Client-side validation gates match server-side validation
- [ ] Existing cron workflows (theme-reveal, ballot-open, tally-winner) unchanged
- [ ] Schema validation passes with new vote data
