## One-Liner

Set up repo structure, GitHub Pages, and working index.html with 80s arcade aesthetic foundation.

## Problem Statement

The repo has no application code — just project config files. We need the foundational directory structure, static HTML shell, and 80s arcade aesthetic CSS so that all subsequent features have a visual and structural home.

## Proposed Solution

1. Create directory structure: `css/`, `js/`, `competitions/`
2. `index.html` — semantic HTML with header (app title + chrome text), main content area (where rounds/themes will live), leaderboard sidebar, and footer
3. `css/styles.css` — dark background, CRT scanline overlay, neon glow text-shadows, Press Start 2P headings, grid layout
4. `js/app.js` — minimal module shell (competition loader placeholder)
5. Google Fonts link for Press Start 2P

## Acceptance Criteria

**Scenario**: Repository structure is established
**Given** the repository has been initialized
**When** the scaffolding is complete
**Then** directories `css/`, `js/`, and `competitions/` exist
**And** `index.html` exists at the root
**And** `css/styles.css` exists
**And** `js/app.js` exists with a basic module structure

**Scenario**: Static site loads with 80s arcade aesthetic
**Given** `index.html` is opened in a browser
**When** the page finishes loading
**Then** there are no console errors
**And** the page has semantic structure: header, main content area, sidebar, and footer
**And** headings render in Press Start 2P pixel font
**And** the background is dark with neon glow text-shadows on key elements
**And** CRT scanline effects are visible

**Scenario**: Zero-dependency static hosting
**Given** the site has no npm dependencies or build step
**When** deployed to GitHub Pages
**Then** all assets load using relative paths
**And** no external dependencies are required beyond Google Fonts
